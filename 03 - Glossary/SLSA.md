# Supply-chain Levels for Software Artifacts (SLSA)

A framework for describing how trustworthy a build is, graded in levels from L0 up to L3 and beyond. Higher levels require signed provenance generated by a hardened build system rather than by the developer.

**Why it matters:** It turns "our builds are secure" into a level you can claim and an auditor can check. L3 in particular rules out a developer injecting artifacts outside the build pipeline.
